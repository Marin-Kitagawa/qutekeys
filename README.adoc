:toclevels: 3

*Advanced keyboard-based web navigation for power users* — a cross-browser (Chrome + Firefox, Manifest V3) extension combining the philosophy of https://qutebrowser.org[qutebrowser] and https://github.com/brookhong/Surfingkeys[SurfingKeys] into one unified tool with switchable keymap profiles, a `:` command palette, full modal navigation, and a Modern Glass UI.

---

== What it is / What it solves

=== The problem

Modern browsers require a mouse for almost everything.
Two excellent keyboard-navigation tools exist — qutebrowser and SurfingKeys — but they split the community: qutebrowser is a standalone Qt/Python browser (not an extension), so it cannot run inside Chrome or Firefox.
SurfingKeys is an extension but uses its own key vocabulary.
Power users who switch between browsers, or who use browser-specific tools, must maintain separate muscle memory for each.

=== How QuteSurf solves it

QuteSurf ships one extension that loads into both Chrome and Firefox (Manifest V3) and offers *three hot-swappable keymap profiles*:

* `qute` — faithful to qutebrowser default bindings
* `surfingkeys` — faithful to SurfingKeys default bindings
* `hybrid` — curated best-of-both (default)

A single *command registry* is the spine: every action is a named command.
Profiles are just `keySequence → commandName` maps.
The `:` key opens a command palette (omnibar) that exposes every command by name, making the tool discoverable.

.Honest scope note
[NOTE]
A WebExtension cannot own the browser chrome, replace the native URL bar, render tabs natively, or run qutebrowser's Python config files.
These capabilities are approximated with extension-level equivalents (omnibar overlay, tabs API, unified JS config).

== Features

=== Modes (mode stack, qutebrowser semantics)

* *Normal* — keyboard navigation active
* *Insert* — key events passed through to the page; `<Escape>` returns to Normal
* *Visual / Caret* — vim-like text selection with `h/j/k/l/w/b/0/$` motions
* *Hints* — link/element selection overlay; `<Escape>` cancels
* *Command* — omnibar / command palette open
* *Find* — in-page text search bar active
* *PassThrough* — all keys forwarded to the page until `<Escape>`

=== Hints

* `hint` — follow links and buttons
* `hint-newtab` — open in a new tab
* `hint-yank` — copy URL or text of element
* `hint-hover` — trigger mouseover
* `hint-input` — focus an input field
* `hint-download` — download a link
* `hint-images` — open selected image in a new tab
* `hint-multi` — follow multiple targets without closing hint mode
* `hint-text` — copy visible text of selected element

Hint labels render as violet gradient chips (`#7c5cff` → `#a855f7`) with soft glow in a Shadow-DOM overlay.

=== Omnibar

Fuzzy-matched floating panel with source badges:

* URL + search (resolved via search-engine aliases)
* Bookmarks
* Browser history
* Open tabs
* Commands (`:` invocable commands)
* Marks and quickmarks

=== Navigation and scrolling

Cardinal scroll (`j/k/h/l`), half-page (`Ctrl-D/U`), full-page (`Ctrl-F/B`), scroll to top/bottom (`gg/G`), percentage (`scroll-to-perc`), back/forward (`H/L`), reload (`r`), hard-reload (`R` in qute profile), stop, URL-up (`gu`), URL-root (`gU`), home.

=== Tabs

New, close, clone, pin, mute, move, undo-close, next/prev (wrapping), first, last, go-to-by-index, detach to new window, close-all-others.

=== Find

In-page forward search (`/`), `n`/`N` for next/previous match.
Options: case-sensitive, regex mode.
Glass find bar.

=== Yank / clipboard

Yank URL, page title, Markdown link (`[title](url)`), selection, focused anchor.
Paste-and-go (current tab) and paste-and-go-newtab — clipboard content is scheme-filtered before navigation (dangerous schemes become search queries).

=== Visual / caret mode

Vim-like text selection.
Caret mode places a movable cursor; Visual mode extends selection.
`y` yanks selection to clipboard.

=== Marks and quickmarks

Local marks (URL + scroll position) set and jumped by a one-shot key capture.
Quickmarks: save (`m` + key) and open (`b` + key / `B` for new tab) in qute/hybrid profiles.
`omnibar-marks` lists all marks in the omnibar.

=== Sessions

Named save/restore of the current window's tabs:
`:session-save work`, `:session-load work`, `:session-list`, `:session-delete work`.

=== Search-engine aliases

Seven built-in aliases (see <<search-engine-aliases>>).
Use in the omnibar: `g cats` searches Google for "cats".
Add custom aliases via the JS settings API or `:set`.

=== Proxy / PAC

Generate and apply a PAC script from per-host rules:
`proxy-set`, `proxy-clear`, `proxy-toggle-host`.
Requires the `proxy` browser permission (already declared).

=== Per-domain userscripts (Greasemonkey-lite)

Store JS snippets with `@match` / `@name` metadata via the options page.
Matched scripts are injected into the page world at `document_idle`.
They run with page-level privileges, not extension privileges.

=== Embedded ACE Vim editor

`edit-with-vim` opens any focused `<input>`, `<textarea>`, or `contenteditable` in an ACE editor with Vim keybindings, rendered in the Shadow-DOM host.
ACE loads as a ~571 KiB lazy chunk on first use.

=== Real Neovim via native messaging

`edit-with-nvim` uses the `com.qutesurf.nvim` native-messaging host to open the field in a real Neovim instance.
Falls back to the ACE editor if the host is not installed.
See <<embedded-vim-editor-and-neovim>>.

=== Unified configuration

Three ways to configure, all writing to the same `chrome.storage.local` store:

. `:set` / `:bind` / `:unbind` at runtime in the command palette
. JS settings API in the options page (sandboxed iframe)
. GUI options page (profile dropdown + settings table + JS editor)

=== Generated help / cheatsheet

`?` or `:help` opens a searchable cheatsheet page built from the command registry and the active profile's bindings, grouped by category.

=== Modern Glass UI

Translucent blurred panels, centered floating command palette, violet accent (`#7c5cff`), soft shadows, rounded corners, dark theme default, themeable via CSS variables.

== Install

=== Prerequisites

* https://nodejs.org[Node.js] ≥ 18
* npm ≥ 9

=== Build from source

[source,shell]
----
git clone <repo-url> qutesurf
cd qutesurf
npm install
npm run build
----

This produces `dist/chrome/` and `dist/firefox/`.

=== Load in Chrome

. Open `chrome://extensions`
. Enable *Developer mode* (toggle, top-right)
. Click *Load unpacked*
. Select the `dist/chrome` directory

=== Load in Firefox

. Open `about:debugging`
. Click *This Firefox*
. Click *Load Temporary Add-on…*
. Select `dist/firefox/manifest.json`

[NOTE]
The Firefox build is temporary (unloaded on browser restart).
For persistent installation, sign and distribute a `.xpi` via the standard Firefox AMO process.

=== Produce distributable zips

[source,shell]
----
npm run package
----

Produces `qutesurf-chrome.zip` and `qutesurf-firefox.zip` in the project root.

== Keymap profiles

Three built-in profiles are included.
The default is `hybrid`.

|===
| Profile | Lineage | Description

| `qute`
| qutebrowser
| Faithful translation of qutebrowser's default normal-mode bindings.

| `surfingkeys`
| SurfingKeys
| Faithful translation of SurfingKeys' default bindings.

| `hybrid`
| Both
| Curated best-of-both: qutebrowser backbone with SurfingKeys aliases where they do not collide.
|===

=== Switching profiles

* *Options page dropdown* — `chrome://extensions` → QuteSurf → Options
* *Popup quick-switch* — click the QuteSurf toolbar icon
* *Command palette* — `:profile qute` / `:profile surfingkeys` / `:profile hybrid`

== The command palette (`:`)

Pressing `:` (or running `cmdline`) opens the command palette — a centered floating omnibar pre-filtered to the *commands* source.
Type any command name (fuzzy-matched) and press `Enter` to run it.

=== Syntax

[source]
----
:<command> [args...] [-flag] [--longflag]
----

* *Positional args* — passed in order to the command handler
* *Short flags* — single letter, e.g. `-t` (new-tab modifier)
* *Long flags* — e.g. `--rapid`
* *Quoted args* — `"my session name"` is treated as one argument
* *Command chaining* — `;;` runs multiple commands in sequence

=== Examples

[source]
----
:open https://example.com           ← navigate to URL
:open -t example.com                ← open in new tab
:tab-close                          ← close current tab
:session-save work                  ← save a named session
:session-load work                  ← restore a named session
:set hintcharacters asdfg           ← change hint charset at runtime
:set smoothscroll false             ← disable smooth scrolling
:bind normal gh open https://example.com  ← add a keybinding
:unbind normal gh                   ← remove a keybinding
:profile qute                       ← switch to qute profile
:proxy-set *.example.com "PROXY 127.0.0.1:8080"
:proxy-clear
:tab-close ;; open example.com      ← chain two commands
----

== Configuration

There are three configuration paths, all reading from and writing to the same `chrome.storage.local` key (`qutesurf:config`).

=== (a) `:set` / `:bind` / `:unbind` at runtime

Open the command palette (`:`) and type commands directly.

.`:set` options (from `src/core/config.js` `DEFAULTS`)

|===
| Option | Default | Description

| `hintcharacters` | `asdfg` | Characters used to generate hint labels
| `scrollstep` | `70` | Pixels per cardinal-direction scroll step
| `smoothscroll` | `true` | Enable CSS smooth-scroll behavior
| `findcasesensitive` | `false` | Case-sensitive in-page find
| `defaultsearchengine` | `g` | Default search engine alias used when input is not a URL
|===

Values are coerced: `"true"` / `"false"` become booleans; numeric strings become numbers; everything else stays a string.

=== (b) JS settings API

The options page includes a JS editor that runs in a sandboxed `<iframe>` (opaque origin, no `chrome.*` access).
Write JavaScript using the SurfingKeys-compatible API:

[source,javascript]
----
// Bind keys
mapkey('gh', 'open home page', 'omnibar-open');
imapkey('<Ctrl-e>', 'edit-with-vim');
vmapkey('Y', 'yank-selection');

// Remove a binding
unmap('v');

// Set options in bulk
settings({
  hintcharacters: 'qwerty',
  smoothscroll: false,
});

// Add a custom search engine alias
aliases('npm', 'https://www.npmjs.com/search?q=%s');

// Hints helpers (SurfingKeys compatibility)
Hints.setCharacters('asdfjkl;');
Hints.style('font-size: 12pt;');   // no-op stub

// Visual helper (stub)
Visual.style('color', 'red');      // no-op stub
----

.Available JS API functions
[cols="1,3"]
|===
| Function | Description

| `mapkey(seq, annotation, cmd)` or `mapkey(seq, cmd)` | Bind in Normal mode
| `imapkey(seq, cmd)` | Bind in Insert mode
| `vmapkey(seq, cmd)` | Bind in Visual mode
| `map(newKey, oldKey)` | Remap (no-op-safe stub; does not crash existing SurfingKeys scripts)
| `unmap(seq, mode?)` | Remove a Normal-mode binding (or specify mode)
| `iunmap(seq)` | Remove an Insert-mode binding
| `vunmap(seq)` | Remove a Visual-mode binding
| `unmapAllExcept(seqs, mode)` | No-op-safe stub (for SurfingKeys script compatibility)
| `settings(obj)` | Set multiple options at once
| `aliases(alias, urlTemplate)` | Add a custom search-engine alias
| `Hints.setCharacters(chars)` | Set hint characters (delegates to `settings`)
| `Hints.style(css, type)` | No-op stub (SurfingKeys compatibility)
| `Visual.style(element, css)` | No-op stub (SurfingKeys compatibility)
|===

[NOTE]
The sandbox runs in an `<iframe sandbox="allow-scripts">` with an opaque/null origin.
`'unsafe-eval'` is scoped *only* to that frame by the `sandbox` CSP directive.
The privileged extension pages have a strict `script-src 'self'` CSP with no `unsafe-eval`.

Access via `chrome://extensions` → QuteSurf → Options (or the popup).

The page provides:

* *Profile dropdown* — select `qute`, `surfingkeys`, or `hybrid`
* *Options table* — view and edit all `:set` options from DEFAULTS
* *JS editor textarea* — paste a settings script and click Save

== Sessions, proxy, and userscripts

=== Sessions

Sessions save and restore all tabs in the current window.

[source]
----
:session-save work            ← save current tabs as "work"
:session-load work            ← open all saved tabs
:session-list                 ← list all saved session names
:session-delete work          ← delete a session
----

Sessions are stored in `chrome.storage.local` and survive browser restarts.

=== Proxy / PAC

QuteSurf generates a PAC script from per-host rules applied via `chrome.proxy.settings`.

[source]
----
:proxy-set *.corp.example.com "PROXY 10.0.0.1:3128"
:proxy-set *.internal "SOCKS5 127.0.0.1:1080"
:proxy-toggle-host example.com       ← toggle proxy on/off for one host
:proxy-clear                         ← remove all rules; restore DIRECT
----

The `cp` key (`proxy-toggle-host`) in the surfingkeys and hybrid profiles uses the first stored rule's proxy as the default proxy for the toggled host.

=== Userscripts

Userscripts are stored via the options page and injected into matching pages at `document_idle`.

[source,javascript]
----
// @name    My Script
// @match   https://example.com/*
console.log('Hello from QuteSurf userscript');
----

[source]
----
:userscript-list              ← print all installed script names to the console
:userscript-remove My Script  ← remove by name
:userscript-add               ← prints guidance to open the options page
----

[WARNING]
Userscripts run in the *page world* with page-level privileges, not extension privileges.
They cannot access `chrome.*` APIs.
Install only scripts you trust, as they execute on every matching page.

== Embedded Vim editor and Neovim

=== Embedded ACE editor (`edit-with-vim`)

`edit-with-vim` opens any focused `<input>`, `<textarea>`, or `contenteditable` element in a full ACE editor with Vim keybindings, rendered inside the Shadow-DOM overlay.
No configuration required.
The ACE bundle (~571 KiB) loads lazily on first use.

Available in both Insert and Normal modes.
Default binding: `<C-i>` in insert mode (set per profile).

=== Real Neovim (`edit-with-nvim`)

`edit-with-nvim` connects to a locally-installed native-messaging host (`com.qutesurf.nvim`) to open the field in a real Neovim instance.
If the host is not installed, it falls back to the embedded ACE editor automatically.

[IMPORTANT]
The current host implementation in `native-host/host.js` is an *echo stub* — it sends text back unchanged.
Real Neovim editing requires wiring `nvim --embed` via msgpack-rpc; see the `NEOVIM INTEGRATION POINT` comment in `host.js` for the integration guide.

==== Installing the native host

See `native-host/README.md` for full platform-specific instructions.
Summary:

. Edit `native-host/com.qutesurf.nvim.json` — set `path` to the absolute path of `host.js` (or a `.bat` wrapper on Windows) and `<EXTENSION_ID>` to your extension ID.
. Register the manifest:
** *Windows (Chrome):* add a registry key at `HKCU\Software\Google\Chrome\NativeMessagingHosts\com.qutesurf.nvim` pointing to the JSON file.
** *macOS (Chrome):* copy to `~/Library/Application Support/Google/Chrome/NativeMessagingHosts/`.
** *Linux (Chrome):* copy to `~/.config/google-chrome/NativeMessagingHosts/`.
** *Firefox (all platforms):* use the `HKCU\Software\Mozilla\NativeMessagingHosts\` registry path (Windows) or `~/.mozilla/native-messaging-hosts/` (Linux/macOS).
. Reload the extension (or restart the browser).

[NOTE]
Node.js ≥ 18 must be in the PATH accessible to the browser process.
On macOS/Linux, if Node is installed via a version manager, you may need to use the absolute path (e.g., `/usr/local/bin/node`) in your `.bat` / wrapper script.

== Architecture

[source]
----
┌─ UI Overlays (Modern Glass, Shadow-DOM isolated) ─────────────┐
│  command palette/omnibar · hint labels · statusline ·          │
│  find bar · which-key cascade · visual feedback                │
├─ Content Scripts ─────────────────────────────────────────────┤
│  hints · scroll · visual/caret · find · marks · yank ·         │
│  embedded Vim editor · DOM glue                                 │
├─ Mode Manager (mode stack)                                     │
├─ Keymap + Profile layer  (qute | surfingkeys | hybrid)        │
├─ Command Registry  (single source of truth: named commands)   │
├─ Background Service Worker ────────────────────────────────────┤
│  tabs/windows · history+bookmark queries · sessions ·          │
│  proxy/PAC · downloads · native-messaging (nvim) ·             │
│  settings store                                                 │
└─ Config system (unified store; :set/:bind + JS API + options) ─┘
----

=== Command Registry (`src/core/registry.js`)

The spine of the extension.
Every action is a `CommandRegistry` entry with a canonical `name`, optional `aliases`, `description`, `args`, `context` (`background` | `content`), `modes`, and a `handler`.
Profiles, key handlers, the omnibar command source, and `:help` all reference this registry.

=== Profile / keymap layer (`src/profiles/`, `src/core/keymap.js`, `src/core/trie.js`)

A profile is a plain `{ name, bindings: { mode: { keySeq: "command" } } }` data object.
Three built-in profiles are registered in `src/profiles/index.js`, which also exports the `CANONICAL_COMMANDS` closed vocabulary that profile bindings are validated against.

Key-sequence resolution is trie-based with:

* Configurable timeout for ambiguous prefixes (triggers the which-key cascade)
* Numeric count prefix support (e.g. `3j` scrolls 3 steps)
* Per-mode key maps; user overrides from config merge on top

=== Mode Manager (`src/core/modes.js`)

A `ModeStack` array with a non-poppable base Normal mode.
`enter(mode)` pushes; `leave()` / `<Escape>` pops.
Subscribers receive `(newMode, prevMode)` notifications; the statusline and key handler both subscribe.

=== Content scripts (`src/content_scripts/`)

Ported and restyled from SurfingKeys: hints engine, scroll helpers, visual/caret mode, find, marks, yank/clipboard, embedded ACE editor.
All Glass UI is rendered into a closed Shadow-DOM host element injected into the page, isolating QuteSurf styles from the page's own CSS.

=== Background service worker (`src/background/index.js`)

MV3 event-driven worker.
A `makeRouter` dispatches incoming `{type:'command', name, args}` messages from content scripts to their background-context handlers.
The router enforces `cmd.context === 'background'` — content-context commands are rejected at this boundary.
All long-lived state is in `chrome.storage`; the worker reconnects on wake.

=== Unified config (`src/core/config.js`, `src/pages/options.js`, `src/pages/sandbox.js`)

`chrome.storage.local` key `qutesurf:config` holds `{ options, userBindings, activeProfile }`.
The `Config` class provides `load()`, `get(key)`, `set(key, value)`, `bind(mode, seq, cmd)`, `unbind(mode, seq)`, `setProfile(name)`.
The options page's JS editor runs in `src/pages/sandbox.js` (opaque-origin iframe) and communicates structured mutations via `postMessage`.

For full design rationale see `docs/superpowers/specs/2026-06-14-qutesurf-extension-design.md` and `docs/superpowers/plans/2026-06-14-qutesurf-extension.md`.

== Security

QuteSurf underwent a full security audit (`docs/SECURITY-AUDIT.md`).
All High and Critical findings are resolved.
As of the audit: 37 test suites / 121 tests pass, including 12 security-specific regression tests.

=== Security posture summary

|===
| Area | Posture

| User JS evaluation
| Runs in a dedicated `src/pages/sandbox.html` — opaque/null origin, no `chrome.*` access, `'unsafe-eval'` scoped only to the sandbox CSP directive. Privileged pages have strict `script-src 'self'`.

| Content-Security-Policy
| `extension_pages`: `script-src 'self'; object-src 'self'; base-uri 'none'`. Chrome additionally has a `sandbox` directive. No inline scripts; no remote scripts.

| Trust-boundary enforcement
| Background router (`makeRouter`) rejects any command whose `context !== 'background'`; content-only commands cannot be invoked through the router.

| Dangerous-scheme navigation
| Central `isSafeNavUrl()` allowlist (`http/https/ftp/mailto/about`) enforced at every navigation sink: omnibar open, paste-and-go, hints newtab/download, background `tab-new`, background `download-url`.

| `web_accessible_resources`
| Narrowed to `pages/help.html`, `pages/help.css`, `pages/help.js`, `chunks/*`. `use_dynamic_url: true` on Chrome rotates URLs per session. Options/popup/sandbox pages are not web-accessible.

| DOM rendering of untrusted strings
| Statusline, which-key, hints, find-marks use `textContent` / `createTextNode` exclusively. Omnibar `innerHTML` sink HTML-escapes every text slice before concatenation. Verified by regression tests (`tests/content/omnibar-xss.test.js`).

| Native messaging host
| No `exec` / `shell:true`. Frame-size cap of 64 MiB. Structural validation (type must be a string) before handling. Error-resilient: bad frames are refused and the buffer reset.

| Userscripts
| Run in the page world (page privileges, not extension privileges). Source is extension-local config only; pages cannot trigger injection.
|===

=== Permissions

|===
| Permission | Justification

| `tabs`
| Tab commands (`tab-new`, `tab-close`, `tab-next`, …) and the omnibar tabs source.

| `bookmarks`
| `bookmark-search` for the omnibar bookmarks source (read-only).

| `history`
| `history-search` for the omnibar history source (read-only).

| `storage`
| Persist config, profile, user bindings, userscripts, sessions, and proxy state in `chrome.storage.local`.

| `clipboardWrite`
| Yank commands copy URL / title / Markdown link / selection.

| `clipboardRead`
| `paste-and-go` and `paste-and-go-newtab` read the clipboard. Pasted content is scheme-filtered before navigation.

| `downloads`
| `hint-download` and `download-url`; URLs are scheme-checked before passing to the downloads API.

| `sessions`
| `tab-undo` (reopen closed tab) and session save/load/list.

| `proxy`
| `proxy-set` / `proxy-clear` / `proxy-toggle-host` generate and apply a PAC script.

| `nativeMessaging`
| Optional Neovim editing host (`connectNative('com.qutesurf.nvim')`); graceful ACE fallback when absent.

| `host_permissions: <all_urls>`
| The extension operates on every page; the content script must match all URLs. Cannot be scoped without removing core functionality.
|===

Removed (unused): `scripting`, `topSites` (see QS-06 in `docs/SECURITY-AUDIT.md`).

== Browser support and limitations

|===
| Target | Status

| Chrome / Chromium (MV3) | Primary target
| Firefox (MV3, ≥ 121.0) | Supported; `browser_specific_settings.gecko.id = qutesurf@local`

=== Known limitations

* The extension cannot own the browser chrome, replace the native URL bar, or render tabs natively — these are outside the WebExtension API surface.
* `reload-hard` (cache bypass) relies on `location.reload()` because `location.reload(true)` is deprecated in modern browsers; true hard-reload is not accessible to extensions.
* The Neovim integration is currently an echo stub.
Real editing requires wiring `nvim --embed` via msgpack-rpc and installing the native host.
See `native-host/host.js` (`NEOVIM INTEGRATION POINT` comment) and `native-host/README.md`.
* Puppeteer e2e tests automatically skip when no browser binary is available.
* ACE editor loads as a ~571 KiB lazy bundle chunk on the first `edit-with-vim` invocation.
* The `home` command navigates to the origin root rather than a configurable homepage (no homepage config yet).
